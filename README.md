# Live Environmental Intelligence Engine

A containerized, stream‑based prototype for detecting environmental pollution anomalies in real time. Data is generated by a configurable simulator, processed via the Pathway streaming engine, and classified by an adaptive isolation forest.

## Overview

The system simulates sensor readings (PM2.5, CO₂, temperature) with drift and occasional spikes, streams the events through Pathway, and flags outliers using an IsolationForest model that retrains periodically on a sliding window. It is built with clean separation between configuration, simulation, modeling, and orchestration layers.

## Objective

Demonstrate a production‑ready streaming anomaly detection engine that can be extended to real IoT data. The prototype is designed for easy configuration, reproducibility, and container deployment.

## Architecture

```
Config -----+
			|
Simulator ---+--> Pathway stream --> Model ---> Alerts/console
```

- **Config layer** (`src/config.py`)
- **Simulation layer** (`src/data_simulator.py`)
- **Model layer** (`src/anomaly_model.py`)
- **Streaming/orchestration** (`src/main.py`)

## Current Status

All core modules are implemented with configuration-driven parameters and adaptive retraining. The project is containerized and ready to run locally or in Docker.

## Notes

- Uses [Pathway](https://www.pathway.com/) for reactive streaming (Linux/macOS only).
- Simulator supports drift and configurable spike probability.
- Model retrains every `RETRAIN_INTERVAL` samples using the last `BUFFER_SIZE` readings.

## Setup

```bash
git clone <repo-url>
cd live-environmental-intelligence-engine

# create and activate virtualenv
python3 -m venv venv
source venv/bin/activate

# install dependencies
pip install --upgrade pip
pip install -r requirements.txt

# run the streaming engine
python -m src.main
```

## Docker

Build and run via Docker:

```bash
docker build -t environmental-engine .
docker run environmental-engine
```
